Multi-Agent Orchestration — First Question

How should the user trigger parallel agents?

A

Automatic — Agent decides

The main agent analyzes the task and spawns sub-agents when it detects parallelizable work. User just types a message as normal.

Pros

  • Zero friction
  • Smart decomposition

Cons

  • Less user control
  • Harder to debug
B

Explicit — User directs it

User explicitly asks for parallel work, e.g. "research X, Y, and Z in parallel" or uses a special syntax like /parallel. Agent only spawns when told to.

Pros

  • Predictable
  • User always in control

Cons

  • Requires user to know the feature exists
C

Both — Auto with override

Agent auto-detects parallel opportunities, but user can also force it or suppress it. Best of both but more complex to build.

Pros

  • Flexible
  • Grows with user familiarity

Cons

  • More surface area to implement correctly
