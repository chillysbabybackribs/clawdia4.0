Data Model & CLI Commands

What gets stored and what the AI can call

calendar_events table (calendar.sqlite)
-- schema
id TEXT PRIMARY KEY -- uuid
title TEXT NOT NULL
date TEXT NOT NULL -- ISO 8601: 2026-03-21
start_time TEXT -- HH:MM, nullable (all-day if null)
duration INTEGER -- minutes, nullable
notes TEXT -- optional free text
created_at TEXT NOT NULL
updated_at TEXT NOT NULL
CLI commands — clawdia-cal
add
clawdia-cal add "Title" --date 2026-03-21 [--time 10:00] [--duration 60] [--notes "..."]
→ prints JSON: {"id":"...","title":"...","date":"..."}

list
clawdia-cal list [--date 2026-03-21] [--from 2026-03-01] [--to 2026-03-31]
→ prints JSON array of events (no args = today)

update
clawdia-cal update <id> [--title "..."] [--date ...] [--time ...] [--duration ...] [--notes "..."]
→ prints updated event JSON

delete
clawdia-cal delete <id>
→ prints {"deleted":true,"id":"..."}

get
clawdia-cal get <id>
→ prints single event JSON

AI context injection

Each time the AI starts a run, the system prompt is prefixed with today's events via clawdia-cal list (no args = today). This gives it passive awareness — it knows your schedule without you having to tell it.

Example injection:
Today is Friday 2026-03-20. You have 2 events: 10:00 Team standup (30m), 14:00 Design review (60m).
