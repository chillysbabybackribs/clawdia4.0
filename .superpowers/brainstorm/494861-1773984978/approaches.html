How should the AI integrate with the calendar?

Three approaches — pick the one that fits best

A

Passive Context Awareness

The AI reads the calendar automatically before every message — it knows what's scheduled and uses that context to give smarter answers. You never have to mention your schedule explicitly.


Pros

  • Zero friction — always aware
  • No new tools or IPC needed beyond a read channel
  • Simple to build and reason about

Cons

  • AI can't create or edit events
  • Calendar data just sits in system prompt context
B

Agent Tool Control

The AI gets dedicated calendar tools: calendar_read, calendar_create_event, calendar_update_event, calendar_delete_event. It calls these when needed, like it calls shell_exec or file_read today.


Pros

  • Full read/write control
  • Fits naturally into existing tool dispatch pattern
  • AI can proactively suggest and create events

Cons

  • More IPC channels to wire up
  • Needs approval flow for destructive actions (delete)
C

Hybrid — Context + Tools + Proactive Suggestions

Calendar state is always injected into context and the AI has full tool control. On top of that, the AI can proactively surface scheduling suggestions inline in chat — "Want me to block 2 hours for that?" — with one-click confirm buttons.


Pros

  • Most powerful UX — AI feels truly calendar-aware
  • Inline suggestions reduce friction for common actions
  • Covers read, write, and proactive scheduling

Cons

  • Most to build
  • Inline suggestion UI needs new ChatPanel components
